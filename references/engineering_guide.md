# 工科类论文写作指南

## 一、工科论文结构规范

### 1.1 标准章节设置

```
第1章 绪论
  1.1 研究背景与意义
  1.2 国内外研究现状
  1.3 研究内容与创新点
  1.4 论文组织结构

第2章 相关技术/理论基础
  2.1 问题定义与数学描述
  2.2 相关技术综述
  2.3 本章小结

第3章 方法论/算法设计
  3.1 整体框架设计
  3.2 核心算法/模块A
  3.3 核心算法/模块B
  3.4 算法复杂度分析
  3.5 本章小结

第4章 实验设计与结果分析
  4.1 实验环境与数据集
  4.2 评价指标
  4.3 对比实验结果
  4.4 消融实验
  4.5 参数敏感性分析
  4.6 本章小结

第5章 结论与展望
  5.1 工作总结
  5.2 创新点总结
  5.3 不足与展望
```

### 1.2 工科论文核心要素

| 要素 | 要求 | 常见错误 |
|------|------|---------|
| **问题定义** | 数学化描述，明确输入输出 | 问题描述模糊，缺乏形式化定义 |
| **技术路线** | 流程图+文字说明 | 只有图没有文字解释 |
| **算法伪代码** | 符合规范，包含复杂度分析 | 伪代码过于具体或过于抽象 |
| **实验对比** | 与SOTA方法+传统方法对比 | 只和自己比或只比一种方法 |
| **消融实验** | 验证每个模块的贡献 | 缺少消融实验或设计不合理 |

## 二、技术路线图规范

### 2.1 系统架构图

**基本要素**：
```yaml
必须包含:
  - 输入层: 明确数据类型和维度
  - 处理层: 各模块名称和连接关系
  - 输出层: 明确输出内容和形式
  
标注要求:
  - 模块名: 简洁明了，英文或中文统一
  - 箭头: 数据流向清晰，可加标注说明变换
  - 颜色: 不超过3种，用于区分不同功能块
```

**示例结构**：
```
输入图像 [224×224×3]
    ↓
特征提取模块 (CNN Backbone)
    ↓ 特征图 [14×14×512]
注意力模块 (Attention Mechanism)
    ↓ 加权特征
分类头 (Classification Head)
    ↓
输出: 类别概率分布 [N_class]
```

### 2.2 算法流程图

**绘制规范**：
- 开始/结束：圆角矩形
- 处理步骤：矩形
- 判断条件：菱形
- 输入/输出：平行四边形
- 流向线：带箭头的直线

## 三、算法伪代码规范

### 3.1 伪代码格式

```latex
\begin{algorithm}
\caption{算法名称}
\label{alg:xxx}
\begin{algorithmic}[1]
\Require 输入参数说明
\Ensure 输出结果说明
\State 初始化...
\For{循环条件}
    \State 执行操作
    \If{判断条件}
        \State 条件操作
    \EndIf
\EndFor
\State \Return 返回值
\end{algorithmic}
\end{algorithm}
```

### 3.2 伪代码编写原则

1. **抽象度适中**：不涉及具体编程语言语法，但保留关键操作
2. **注释充分**：复杂操作需加注释说明
3. **复杂度标注**：时间复杂度（如O(n²)）和空间复杂度

## 四、实验设计规范

### 4.1 数据集描述模板

```markdown
## 数据集信息表

| 数据集 | 样本数 | 类别数 | 图像尺寸 | 特点 |
|--------|-------|-------|---------|------|
| Dataset A | 50,000 | 10 | 32×32 | 小规模，用于初步验证 |
| Dataset B | 100,000 | 100 | 224×224 | 大规模，接近实际应用 |

## 数据划分
- 训练集：70%（35,000张）
- 验证集：15%（7,500张）
- 测试集：15%（7,500张）

## 数据预处理
- 归一化：均值[0.485, 0.456, 0.406]，标准差[0.229, 0.224, 0.225]
- 数据增强：随机裁剪、水平翻转、颜色抖动
```

### 4.2 评价指标

**分类任务**：
- 准确率（Accuracy）
- 精确率（Precision）
- 召回率（Recall）
- F1分数
- AUC-ROC

**检测/分割任务**：
- mAP（mean Average Precision）
- IoU（Intersection over Union）
- Dice系数

**必须报告**：
- 均值 ± 标准差（多次运行）
- 最好/最差性能
- 与基线方法的性能差距（%）

### 4.3 消融实验设计

**标准消融实验表**：

| 模块A | 模块B | 模块C | 准确率(%) | 参数量(M) | 推理时间(ms) |
|-------|-------|-------|----------|----------|-------------|
| ✗ | ✗ | ✗ | 82.3 | 10.2 | 15 |
| ✓ | ✗ | ✗ | 85.1(+2.8) | 11.5 | 18 |
| ✓ | ✓ | ✗ | 87.4(+2.3) | 13.8 | 22 |
| ✓ | ✓ | ✓ | 89.2(+1.8) | 15.1 | 25 |

**解读示例**：
> 消融实验结果表明，模块A对性能提升贡献最大（+2.8%），这验证了{某假设}；模块B的引入带来了{某方面}的改进，但代价是{某代价}；模块C虽然提升幅度较小（+1.8%），但考虑到其带来的{某优势}，整体仍是值得的。

## 五、图表规范

### 5.1 实验结果表

**对比实验表模板**：

```markdown
表4-1 不同方法在XXX数据集上的性能对比（%）

| 方法 | 年份 | 准确率 | 精确率 | 召回率 | F1分数 |
|------|------|-------|-------|-------|-------|
| 传统方法A | 2018 | 78.5 | 76.2 | 79.1 | 77.6 |
| 传统方法B | 2019 | 81.2 | 79.8 | 80.5 | 80.1 |
| 深度学习方法A | 2021 | 86.3 | 85.1 | 84.9 | 85.0 |
| 深度学习方法B | 2022 | 88.1 | 87.3 | 86.8 | 87.0 |
| 本文方法 | 2024 | **90.5** | **89.8** | **89.2** | **89.5** |

注：加粗表示最优结果；所有实验重复5次，报告均值。
```

### 5.2 收敛曲线图

**必须包含的信息**：
- 横轴：迭代次数/epoch
- 纵轴：损失值/准确率
- 多条曲线：训练集+验证集
- 图例：清晰标识各曲线含义
- 阴影区域：多次运行的标准差范围

## 六、写作要点

### 6.1 语态与句式

**推荐语态比例**：
- 被动语态：≥60%
- 主动语态：≤40%

**常用被动表达**：
- "实验结果表明..."
- "数据被收集于..."
- "该方法被应用于..."
- "性能被评估使用..."

### 6.2 关键段落模板

**创新点陈述**：
```
与以往研究相比，本文的主要贡献包括：
（1）提出了XXX方法，有效解决了XXX问题；
（2）设计了XXX模块，在XXX方面取得了显著提升；
（3）在多个公开数据集上验证了方法的有效性，达到了SOTA性能。
```

**实验结论**：
```
综上所述，实验结果表明本文提出的XXX方法在XXX任务上取得了良好的性能。
具体来说，在Dataset A上，我们的方法相比基线方法提升了X.XX%的准确率；
在Dataset B上，提升幅度达到了X.XX%。消融实验进一步验证了各模块的有效性，
其中XXX模块对性能提升贡献最大。这些结果充分说明了本文方法的有效性。
```

## 七、常见陷阱与规避

### 7.1 实验设计陷阱

| 陷阱 | 问题 | 规避方法 |
|------|------|---------|
| 单一数据集验证 | 泛化能力存疑 | 至少在3个不同数据集上验证 |
| 缺少消融实验 | 无法证明各模块贡献 | 系统性地移除各模块进行对比 |
| 评价指标单一 | 性能评估不全面 | 至少报告3个互补指标 |
| 测试集调参 | 数据泄露 | 严格区分训练/验证/测试集 |
| 未报告随机种子 | 结果不可复现 | 明确报告随机种子设置 |

### 7.2 写作陷阱

| 陷阱 | 问题 | 规避方法 |
|------|------|---------|
| 过度宣称 | "证明了..." | 使用"表明"、"支持"等谨慎表述 |
| 缺乏对比 | 只和自己比 | 与SOTA方法和传统方法都对比 |
| 忽略局限性 | 显得不够严谨 | 专门设置局限性讨论章节 |
| 参数设置不明 | 难以复现 | 所有超参数都在正文中或附录中给出 |
